# 小红书分享流

每篇文章的 frontmatter 都可以写 `xhs` 字段，并在正文里放一行 `<XhsShare />`。发布前在本地预览页面，截取竖版分享卡片，再点按钮复制文案。构建时还会为每篇文章生成一张 4:5 的 SVG 分享卡片，路径类似 `/xhs-cards/hello-blog.svg`。

推荐节奏：

1. 博客写完整推理、链接、实验和参考资料。
2. 小红书只保留一个明确问题、三到五个要点、一个可讨论的结论。
3. 结尾放具体文章链接，把长文作为延伸阅读。
4. 图片优先使用 `<XhsShare />` 的“打开分享卡片”链接；正文里再补一到两张关键图或代码截图。
5. 小红书标题不要照搬论文标题，改成读者能立刻理解的问题或结论。

frontmatter 示例：

```yaml
---
title: 文章标题
date: 2026-06-26
category: papers
tags: [LLM, 论文精读]
description: 一句话摘要。
xhs:
  title: 适合小红书的标题
  summary: 这篇文章最值得带走的一句话。
  # 可选：默认会自动生成当前博客文章链接。
  # url: https://sunzhongkai588.github.io/posts/example
  bullets:
    - 第一个要点
    - 第二个要点
    - 第三个要点
---
```

## 发布前检查

1. 标题是否能在手机截图里完整显示。
2. `summary` 是否是一句话，而不是摘要段落。
3. `bullets` 是否控制在三到五条。
4. 标签是否同时包含主题标签和受众标签，例如 `#论文精读`、`#大模型`。
5. 博客正文是否保留了参考链接，方便从小红书回流后继续阅读。
